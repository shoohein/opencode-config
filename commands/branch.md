description: Create a branch and commit changes

Check the current git status and recent branch naming conventions.

Based on the changes, determine an appropriate branch name.

Create the branch and show the user the following plan for confirmation:

- Branch name
- Files to be committed
- Proposed commit message (following git-commit skill conventions)

Wait for the user's explicit approval before staging and committing.

After approval, stage the changes and commit using the git-commit skill.
